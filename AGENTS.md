# JSON 对比工具 — 设计文档

## 项目概述

一个纯前端静态 JSON 对比工具，无需后端服务，数据完全在浏览器本地处理。用户输入两个 JSON（基准 JSON 和被测试 JSON），工具以并排 Diff 视图直观展示被测试 JSON 相对于基准 JSON 的变化。

## 页面布局

```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤
│  输入基准 JSON 和被测试 JSON，点击"对比"查看差异             │  ← Stats Bar
├────────────────────────────┬────────────────────────────────┤
│  基准 JSON        [有效]   │  被测试 JSON      [有效]       │  ← Panel Headers
│  1  {                      │  1  {                          │
│  2    "name": "Alice",     │  2    "name": "Alice",         │
│  3    "age": 25,      ←红  │  3    "age": 26,      ←绿      │  ← Diff View
│  4    "city": "Beijing",   │  4    "country": "China",      │
│  5    "hobbies": [         │  5    "hobbies": [             │
│  6      "reading",         │  6      "reading",             │
│  7      "music"       ←红  │  7      "travel"      ←黄      │

│  9  }                      │  9  }                          │
├────────────────────────────┴────────────────────────────────┤
```

## 核心功能

### 1. 双栏并排 Diff 视图（Side-by-Side）
- 左右两个面板分别展示基准 JSON 和被测试 JSON 的格式化输出
- 差异直接在对应行上以**背景色**高亮，用户无需在"输入区"和"结果区"之间来回扫视
- 删除行（少了）→ 红色背景
- 新增行（多了）→ 绿色背景
- 修改行（改了）→ 黄色背景
- 未变更行 → 正常显示，无高亮

### 2. 字符级差异（Character-Level Diff）
- 对于被修改的行，不仅整行变色，具体变化的字符还会再用深色块标出
- 删除的字符 → 深红色背景
- 新增的字符 → 深绿色背景
- 基于 LCS（最长公共子序列）算法对 token 级内容做精确对齐

### 3. 视图模式切换
- **并排视图（Side）**：左右双栏对比，默认模式，适合大多数场景
- **内联视图（Inline）**：单栏展示，隐藏右侧面板，适合小屏幕或快速浏览
- 切换按钮在 Header 工具栏中，点击即时切换

### 4. 编辑 ↔ 对比 模式切换
- **对比按钮**：点击"对比"进入 Diff 视图，按钮变为"编辑"
- **回到编辑**：
  - 再次点击"编辑"按钮
  - 点击任意 Diff 结果区域
  - 直接在 textarea 中输入内容
- 切换平滑，不丢失已输入的 JSON 内容

### 5. 实时 JSON 语法校验
- 用户输入时**即时解析** JSON
- 解析成功：面板头部状态显示 "有效"（绿色标签）
- 解析失败：状态显示具体错误信息，例如 "第 3 行：Unexpected token }"（红色标签）
- 行号通过 `JSON.parse` 错误信息中的 `position` 自动推算
- 点击"对比"时也会再次校验，不合法的 JSON 不会进入 Diff 视图

### 6. 同步滚动
- 左右两个面板滚动时自动同步
- 确保两侧对应行始终对齐，方便逐行对比

### 7. 辅助操作
| 按钮 | 功能 |
|------|------|
| 示例 | 加载预设的示例 JSON 到两侧输入框 |
| 清空 | 清空两侧输入框，重置所有状态 |
| 交换 | 交换基准 JSON 和被测试 JSON 的内容 |
| 并排/内联 | 切换视图模式 |
| 对比/编辑 | 进入或退出 Diff 视图 |

### 8. 差异统计
- 顶部统计条显示差异总数
- 分类统计：多了 / 少了 / 改了

## 技术实现

### 技术栈
- **纯静态 HTML**，无框架、无构建工具、无 npm 依赖
- 原生 JavaScript + CSS
- Google Fonts（Noto Sans SC）

### Diff 算法
1. **行级 Diff**：使用 LCS（最长公共子序列）算法对比格式化后的 JSON 行数组
2. **字符级 Diff**：对相邻的删除/新增行对，使用 token 级 LCS 算法做精细对比
   - Token 类型：字符串、数字、布尔值、null、标点符号、空白符

### 核心数据结构
```javascript
// 行级对齐结果
{ aNum: 3, bNum: 3, aLine: '"age": 25,', bLine: '"age": 26,', type: 'mod' }
{ aNum: 4, bNum: null, aLine: '"city": "Beijing",', bLine: null, type: 'del' }
{ aNum: null, bNum: 4, aLine: null, bLine: '"country": "China",', type: 'add' }
```

### 状态管理
- `viewMode`: 'side' | 'inline' — 当前视图模式
- `comparing`: boolean — 是否在 Diff 视图
